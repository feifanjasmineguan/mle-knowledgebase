## Core Concept

The **bias-variance tradeoff** describes a fundamental tension in supervised learning between two sources of model error that pull in opposite directions.

**Total Expected Error = Bias² + Variance + Irreducible Noise**

---

## Definitions

### 🎯 Bias
- **What it is:** Error from incorrect assumptions in the learning algorithm
- **Cause:** Model is too simple to capture the true underlying pattern
- **Symptom:** **Underfitting** — poor performance on *both* training and test data
- **Example:** Fitting a linear model to quadratic data

### 🎲 Variance
- **What it is:** Error from sensitivity to small fluctuations in training data
- **Cause:** Model is too complex, memorizing noise instead of signal
- **Symptom:** **Overfitting** — great training performance, poor test performance
- **Example:** A deep decision tree with no pruning

### 🔇 Irreducible Noise
- Inherent randomness in the data; **cannot be eliminated** by any model

```
  |        \  /   Variance
  |         \/
  |________________________

```

| Model Complexity | Bias | Variance | Result |
|---|---|---|---|
| Too Low | High | Low | Underfitting |
| **Just Right** | **Balanced** | **Balanced** | **Optimal** |
| Too High | Low | High | Overfitting |

## Practical Strategies

### Reducing High Bias (Underfitting)
- Increase model complexity (more layers, more parameters)
- Add more features / better feature engineering
- Reduce regularization strength
- Train longer

### Reducing High Variance (Overfitting)
- **Regularization:** L1 (Lasso), L2 (Ridge), Dropout
- **More training data**
- **Ensemble methods:** Bagging (e.g., Random Forest) reduces variance by averaging
- Early stopping, cross-validation
- Reduce model complexity

### Ensemble Methods & the Tradeoff
| Method | Effect |
|---|---|
| **Bagging** (e.g., Random Forest) | Reduces **variance** |
| **Boosting** (e.g., XGBoost) | Reduces **bias** |
| **Stacking** | Can reduce both |

## Likely Follow-Up Interview Questions

### Conceptual
1. **"How do you diagnose whether your model has high bias or high variance?"**
   > Compare training vs. validation error curves

2. **"Does more data always help?"**
   > It primarily helps with high variance; it doesn't fix high bias

3. **"Can a model have both high bias and high variance simultaneously?"**
   > Yes — e.g., a misspecified complex model on sparse data

### Technical Deep Dives
4. **"How does regularization affect the bias-variance tradeoff mathematically?"**
   > L2 regularization shrinks weights → increases bias, decreases variance

5. **"Why does bagging reduce variance but not bias?"**
   > Averaging independent estimators reduces variance (by ~1/n); bias of each estimator unchanged

6. **"How does cross-validation relate to the bias-variance tradeoff?"**
   > Estimates generalization error; k-fold CV itself has a bias-variance tradeoff (k=n is low bias/high variance)

7. **"Where does dropout fit in this framework?"**
   > Acts as regularization → reduces variance at slight cost to bias
