## What Is the Bias-Variance Tradeoff?

The **bias-variance tradeoff** describes the tension between two sources of model error that pull in opposite directions when choosing model complexity.

### Total Expected Error Decomposition

```
Total Error = Bias² + Variance + Irreducible Noise
```

| Component | Definition | Caused By |
|-----------|-----------|-----------|
| **Bias** | Error from wrong assumptions in the model | Underfitting, oversimplified model |
| **Variance** | Error from sensitivity to small fluctuations in training data | Overfitting, overcomplicated model |
| **Irreducible Noise** | Inherent randomness in the data | Cannot be reduced |

---

## Intuitive Explanation

### High Bias (Underfitting)
- Model is **too simple** to capture the true pattern
- Consistently wrong in the same direction
- Poor performance on **both** training and test sets
- Example: Fitting a linear model to quadratic data

### High Variance (Overfitting)
- Model is **too complex** and memorizes training data
- Very sensitive to which training samples are used
- Great training performance, poor test performance
- Example: A very deep decision tree with no pruning

## Practical Examples

| Model | Bias | Variance |
|-------|------|----------|
| Linear Regression | High | Low |
| Deep Neural Network (unregularized) | Low | High |
| Random Forest | Low-Med | Lower than single tree |
| k-NN (k=1) | Low | High |
| k-NN (k=N) | High | Low |

---

## How to Fix Each Problem

### Reducing High Bias
- Use a more complex model
- Add more features / feature engineering
- Reduce regularization strength
- Train longer (for neural networks)

### Reducing High Variance
- **Regularization**: L1/L2, dropout
- **More training data**
- **Ensemble methods**: Bagging (e.g., Random Forest)
- **Early stopping**
- **Cross-validation** to detect overfitting

## Likely Follow-Up Interview Questions

> 1. **"How does regularization affect the bias-variance tradeoff?"**
> → Regularization *increases* bias slightly but *reduces* variance — often a net win

> 2. **"How do ensemble methods like bagging help?"**
> → Averaging multiple high-variance models reduces variance without substantially increasing bias

> 3. **"What is the double descent phenomenon?"**
> → In modern deep learning, test error can *decrease again* after the classical overfitting regime as model size grows extremely large

> 4. **"How do you diagnose high bias vs. high variance in practice?"**
> → Learning curves: large train-test gap = high variance; both errors high = high bias

> 5. **"Does more data always help?"**
> → Helps with high variance, but cannot fix high bias

> 6. **"How does cross-validation relate to bias-variance?"**
> → Provides an unbiased estimate of test error and helps detect variance problems early
