## Core Concept

The **bias-variance tradeoff** describes a fundamental tension in supervised learning: the two main sources of prediction error pull in opposite directions, and reducing one typically increases the other.

The **expected generalization error** of a model can be decomposed as:

$$\text{Expected Error} = \text{Bias}^2 + \text{Variance} + \text{Irreducible Noise}$$

---

## Definitions

### 🎯 Bias
- Error from **wrong assumptions** in the learning algorithm
- The difference between the **average prediction** of the model and the **true value**
- High bias → model is **too simple** → **underfitting**
- Example: fitting a linear model to a quadratic relationship

$$\text{Bias} = \mathbb{E}[\hat{f}(x)] - f(x)$$

### 📊 Variance
- Error from **sensitivity to fluctuations** in the training set
- How much the model's predictions **change** across different training datasets
- High variance → model is **too complex** → **overfitting**
- Example: a deep decision tree memorizing training data noise

$$\text{Variance} = \mathbb{E}\left[(\hat{f}(x) - \mathbb{E}[\hat{f}(x)])^2\right]$$

### 🔊 Irreducible Noise
- Error from **inherent randomness** in the data
- Cannot be reduced regardless of model choice

## Practical Examples

| Algorithm | Bias | Variance |
|---|---|---|
| Linear Regression | High | Low |
| Deep Neural Network | Low | High |
| Decision Stump | High | Low |
| Deep Decision Tree | Low | High |
| Random Forest | Low | **Lower** (via averaging) |
| Bagging | Same | **Reduced** |
| Boosting | **Reduced** | Slightly increased |

---

## Key Mitigations

### Reducing High Bias
- Use a more complex model
- Add more features / feature engineering
- Reduce regularization

### Reducing High Variance
- Get **more training data**
- Add regularization (L1, L2, dropout)
- Use ensemble methods (bagging, random forests)
- Reduce model complexity
- Cross-validation for model selection

## Likely Follow-Up Interview Questions

### Conceptual
1. **"Can you have both high bias and high variance simultaneously?"**
   - Yes — e.g., a high-degree polynomial trained on a small, noisy dataset in the wrong region

2. **"How does regularization affect bias and variance?"**
   - L1/L2 increases bias but reduces variance; the regularization strength $\lambda$ controls this tradeoff

3. **"Is the bias-variance tradeoff relevant for modern deep learning?"**
   - Partially — the **double descent** phenomenon shows that very overparameterized models can generalize well, challenging classical intuition

### Applied
4. **"Your model has high training accuracy but poor test accuracy. What's likely wrong and how do you fix it?"**
   - High variance / overfitting → more data, regularization, simpler model

5. **"How does cross-validation help with the bias-variance tradeoff?"**
   - Gives a better estimate of generalization error, helping select models at the optimal complexity

6. **"How does ensemble learning (bagging vs. boosting) address this tradeoff?"**
   - **Bagging** reduces variance by averaging independent models
   - **Boosting** reduces bias by sequentially correcting errors

7. **"How does increasing training data affect each component?"**
   - Reduces variance; bias is largely unchanged (it's a property of model family, not data size)
